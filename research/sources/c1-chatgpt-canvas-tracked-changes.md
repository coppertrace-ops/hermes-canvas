# ChatGPT Canvas — tracked changes / diff UX (PRESS)

- URL: https://venturebeat.com/ai/chatgpts-canvas-now-shows-tracked-changes (VentureBeat; direct fetch 403)
- Source type: press. Corroborated by community "six months" retrospective (c1-chatgpt-canvas-retired-medium.md).
- Accessed: 2026-07-13

## Key claims (from headline + corroborating sources)
- OpenAI added a **"tracked changes" / Show changes** view to Canvas so AI edits surface as additions/deletions (word-processor-style redlining) rather than a silent whole-document rewrite. Applies to both documents and code.
- Community retrospective confirms edits were "highlighted in green so you could see exactly what moved," and that this non-destructive, visible-diff behavior was the feature users valued most.

## Relevance to Hermes Canvas
- **Diff legibility is the differentiator.** Users repeatedly praise seeing *exactly what the AI changed* (green highlight / tracked changes) and distrust surfaces that regenerate opaquely. Hermes should default to an **inline, reviewable diff** for every agent edit to the canvas.
- Tracked-changes framing (accept/reject, additions vs deletions) maps cleanly onto a human review mental model — lower cognitive load than a raw before/after dump.
